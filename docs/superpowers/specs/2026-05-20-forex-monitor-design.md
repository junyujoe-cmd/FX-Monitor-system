# 实时外汇监控系统 — 设计文档

## 概述

一个桌面实时外汇监控系统，监控 USDCNY、EURCNY、HKDCNY、JPYCNY 四个货币对，
自动获取市场报价，模拟银行成本价，支持用户录入实际报价并计算银行赚取的 bp。

## 技术栈

- Python 3.x + PyQt6（桌面 GUI）
- SQLite（本地数据存储）
- QTimer + QThread（定时后台抓取数据，不阻塞 UI）
- free exchangerate API + 银行网站爬虫（数据来源）

## 架构

单进程 PyQt6 应用，按功能拆分为独立模块：

```
forex-monitor/
├── main.py              # 入口：启动 QApplication + 主窗口
├── ui/
│   ├── main_window.py   # 主窗口布局（报价条 + 标签页）
│   ├── quote_bar.py     # 顶部浮动报价条组件
│   ├── compare_panel.py # 银行对比表格 + 用户录入面板
│   └── chart_widget.py  # 历史图表组件（PyQtGraph / Matplotlib）
├── fetcher.py           # 数据抓取模块（API + 爬虫，运行在 QThread）
├── database.py          # SQLite 读写接口
├── calculator.py        # 成本价计算 + bp 换算逻辑
├── config.py            # 配置常量（银行列表、货币对、API 地址等）
├── requirements.txt
└── .gitignore
```

## 数据流

```
QTimer (每 60 秒)
  └→ DataFetcher (QThread 后台线程)
       ├→ 免费汇率 API → 市场中间价
       ├→ (可选) 银行官网爬虫 → 银行报价
       └→ 存入 SQLite
            └→ calculator.py 计算成本价 + bp
                 └→ 刷新 UI 显示
```

用户操作流:
```
用户看到报价 → 在录入面板选择 (货币对 + 银行 + 方向 + 报价)
  └→ 系统计算两种成本:
       1. 市场成本价（CFETS 中间价）
       2. 银行自身成本（该银行买价 + 卖价）/ 2
     └→ 显示银行赚了多少 bp
     └→ 存入 SQLite（历史记录）
```

## 买入/卖出价处理

- 浮动报价条显示每个货币对的 买价 | 中间价 | 卖价
- 对比表格顶部有"买入价"/"卖出价"切换开关
- 切换后表格和 bp 计算按对应方向重新计算
- 用户录入时可以选方向（买入或卖出）

## 银行成本价计算

两种成本价同时显示:
1. **市场成本价** — CFETS 中间价（USDCNY.IB）/ 汇率 API 中间价
2. **银行自身成本** — 该银行挂牌（买入价 + 卖出价）/ 2

用户录入报价后，同时显示与两种成本价的 bp 差异。

## 数据存储 (SQLite)

表结构:
- `market_quotes` — 自动抓取的市场报价（时间, 货币对, 买价, 卖价, 来源）
- `bank_quotes` — 自动抓取的银行报价（时间, 银行, 货币对, 买价, 卖价）
- `user_entries` — 用户录入的报价（时间, 货币对, 银行, 方向, 报价, 计算出的 bp）

## 数据来源（分阶段实现）

1. **Phase 1**: 免费公开汇率 API（获取市场中间价）
2. **Phase 2**: 爬取中行、招行、宁波银行、杭州银行官网外汇牌价
3. **Phase 3**: 接入 CFETS（USDCNY.IB）数据
